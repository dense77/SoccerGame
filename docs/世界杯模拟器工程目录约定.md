# 世界杯模拟器工程目录约定

## 1. 文档目标

本文档定义首版工程目录结构和代码落位约束，目的是在真正开始编码前先固定职责边界，避免后续逻辑散落在页面组件里。

## 2. 目录设计原则

- 业务规则与 UI 分离
- 数据访问与领域逻辑分离
- 模块按能力拆分，而不是按页面堆叠
- 测试文件按单元测试和集成测试明确分层

## 3. 建议目录结构

```text
src/
  app/
    providers/
    router/
    store/
  data/
    db/
    repositories/
    seeds/
  domain/
    rules/
    value-objects/
  modules/
    tournament/
    team-management/
    match-engine/
    event-system/
    save-progress/
  types/
  ui/
    pages/
    components/
    view-models/
  test/
    unit/
    integration/
```

## 4. 各目录职责

### 4.1 app

职责：

- 应用启动
- 全局 provider
- 路由和页面装配
- 应用级状态桥接

禁止：

- 写比赛规则
- 写 SQL

### 4.2 data

职责：

- SQLite 初始化
- schema 和迁移
- 仓储实现
- 种子数据导入

禁止：

- 写业务决策规则
- 直接依赖 React 组件

### 4.3 domain

职责：

- 规则参数
- 值对象
- 与框架无关的领域逻辑辅助

适合放置：

- 阵型克制表
- 战术相性表
- 状态枚举
- 评分区间映射

### 4.4 modules

职责：

- 组织核心业务能力

模块划分固定为：

- `tournament`
- `team-management`
- `match-engine`
- `event-system`
- `save-progress`

每个模块内部建议包含：

- `services/`
- `models/`
- `adapters/`
- `tests/` 可选

### 4.5 types

职责：

- 放共享类型定义

限制：

- 只放稳定公共类型
- 不把全部业务实现都堆在这里

### 4.6 ui

职责：

- 页面
- 组件
- 展示层状态组装

禁止：

- 内嵌赛果计算
- 内嵌事件触发规则
- 直接读写 SQLite

### 4.7 test

职责：

- 统一承载跨模块测试入口和测试辅助

结构固定为：

- `test/unit`
- `test/integration`

## 5. 测试目录约定

### 5.1 单元测试

放置原则：

- 测纯逻辑
- 测纯仓储接口
- 测不依赖 UI 的规则计算

应覆盖：

- 比赛规则
- 赛事推进
- 事件筛选与效果执行
- 数据库初始化幂等性
- 存档状态迁移

### 5.2 集成测试

放置原则：

- 测模块联动
- 测仓储与业务联动
- 测 UI 到模块的关键路径

应覆盖：

- 选队到存档创建
- 赛前排阵到比赛结算
- 比赛结果到赛事推进
- 赛后状态到下一场恢复

## 6. 命名约定

- 仓储实现统一放在 `repositories`
- 规则参数统一放在 `domain/rules`
- 共享类型统一放在 `types`
- 模块服务统一使用 `*Service` 结尾
- 仓储接口统一使用 `*Repository` 结尾
- 测试文件统一使用 `.test.ts` 结尾

## 7. 单元测试要求

工程结构相关单元测试主要是静态和约束性测试，至少覆盖：

- 模块服务不依赖 UI 组件
- 数据层不引用页面组件
- 比赛规则参数不出现在页面目录

建议单元测试用例：

- 用例 1：`match-engine` 模块在无 React 环境下可独立实例化
- 用例 2：`data/repositories` 不引用 `ui/pages`
- 用例 3：`ui` 目录下不存在直接执行 SQL 的实现

## 8. 集成测试要求

工程结构相关集成测试至少覆盖：

- 应用启动后可以通过 app -> repository -> module -> ui 完成一条主链路
- 测试环境可以替换仓储实现或种子数据而不影响上层流程

建议集成测试用例：

- 用例 1：最小种子数据下，应用能完成“选队 -> 创建存档 -> 进入主界面”
- 用例 2：用测试仓储替换正式仓储后，比赛流程仍能跑通

## 9. 非目标

本目录约定当前不约束：

- 具体 CSS 文件命名
- 组件视觉风格
- 最终路由路径细节
